# Draft Content in the User's Voice

Produce publish-ready drafts that sound like the user wrote them, not like AI content.

## Your voice profile (user: fill this in — see USER-INSTRUCTIONS.md)

- **Banned words/patterns:** [buzzwords and tics you never use]
- **Rhythm rules:** [paragraph length, sentence rhythm, emoji policy, character limits]
- **Worldview:** [3-5 recurring beliefs]
- **Signature stance:** [your one-line position on your core topic]
- **Real stories drafts may use:** [3-5 true anecdotes with specifics]
- **Content pillars:** [P1 ... P6 named themes]

- **Platforms:** [primary platform first, then adaptation targets — default: LinkedIn primary, Instagram + YouTube adaptations]

If this section is still blank, stop and ask the user to complete it before drafting.

## Before drafting — always

1. Read the voice profile above in full.
2. Get the idea source: a row from the idea bank in ~~spreadsheet (Content Ideas tab), or an idea the user supplies directly.

## Selecting ideas (when drafting a batch)

- Default batch: 3-4 drafts, primary platform first.
- Prioritize: High-priority rows not yet drafted, ideas tied to fresh platform news (timeliness decays), and at least one from a pillar not covered in the previous batch.
- Mix formats: never a batch of all plain text posts. Include at least one visual-first format (carousel, meme, or video script).

## Format decision guide

| Idea type | First choice | Second choice |
|---|---|---|
| Contrarian take | Text post + meme or cover image | Short talking-head video |
| Step-by-step framework | Carousel (6-10 slides, one idea per slide, hook-only slide 1) | Text post with sectioned takeaways |
| Story / troubleshooting narrative | Text post + cover image | Video walkthrough |
| Tool comparison / live test | Long video → short cutdowns → text teaser | Carousel scorecard |
| Quick win / observation | Short text post in a recurring series | Story frames |

## Drafting rules

- Follow the user's post structure for story posts. For contrarian takes, lead with the take in line one, then ground it.
- Every draft must pass the voice check: no banned words, rhythm rules respected, at least one specific detail (number, tool name, real situation), and a closing question.
- Never fabricate stories, client examples, metrics, or events. Pull specifics from the idea bank raw content or the voice profile's real stories, or mark clearly: `[YOU: add real example here]`.
- Each draft ends with: suggested format, creative spec (one line: what visual accompanies it), best posting window, and 2-3 line adaptation notes per secondary platform.

## Output format per draft

```
### Draft N: [Working title]
Pillar: PX | Format: [format] | Post window: [day/time]
[Full post text, ready to paste]
Creative: [one-line spec]
[Platform 2] adaptation: [note]
[Platform 3] adaptation: [note]
```

## After drafting

- Offer to add drafts to the Full Post Drafts tab of ~~spreadsheet and to generate visuals with the `create-post-creative` skill.
- Present all drafts for review. Never post or send anything anywhere without the user's explicit approval.
